# TodoList 需求规格说明书 v2.0

## 1. 项目概述

| 项目 | 说明 |
|------|------|
| 产品名称 | TodoList |
| 产品定位 | 轻量级跨端（Web + 移动端）待办事项管理工具 |
| 技术栈 | HTML/CSS/JS 前端 + Express.js Serverless API + Supabase (PostgreSQL + Auth) |
| 部署平台 | Vercel |
| 认证方式 | GitHub OAuth（Supabase Auth） |

## 2. 功能模块

### 2.1 登录页

- 珊瑚色渐变背景 + 毛玻璃卡片居中布局
- GitHub OAuth 一键登录按钮（SVG 图标 + 文字）
- 未登录时显示登录页，已登录自动跳转首页
- 登录失败 Snackbar 提示错误信息

### 2.2 首页 Tab（Dashboard）

- 时段问候语：上午好（<12点） / 下午好（12-18点） / 晚上好（>18点）
- 当前日期：X月X日 星期X
- 右上角"+"按钮：点击跳转任务页并自动聚焦输入框
- 今日完成率卡片：珊瑚色渐变背景 + SVG 圆形进度环 + 百分比数字 + "X / Y 已完成"
- 四格统计卡片（2x2 网格）：
  - 总待办（全部任务数）
  - 已完成（全部已完成数）
  - 待完成（全部未完成数）
  - 完成率（整体完成百分比）
- 今日待办列表预览（最多 5 条，按 due_date 筛选当天任务）
- "查看全部 >"链接：点击跳转任务 Tab
- 无今日待办时显示空状态："今日暂无待办，好好休息吧"

### 2.3 任务 Tab（全部任务）

#### 添加任务
- 输入框 + "+"按钮，支持 Enter 键提交
- 聚焦输入框展开选项栏（优先级 / 分类 / 截止日期）
- 点击页面其他区域或 ESC 键收起选项栏
- 添加后立即显示在列表中（乐观更新），后台异步同步

#### 搜索与筛选
- 搜索框：关键词模糊匹配，300ms 防抖
- 筛选按钮组：全部 / 未完成 / 已完成

#### 任务列表
- 每个任务项包含：圆形复选框 + 任务内容 + 标签组 + 编辑按钮 + 删除按钮
- 标签组：优先级标签（高/中/低，彩色）+ 分类标签 + 截止日期标签
- 已过期且未完成的任务标红显示"⚠"
- 已完成任务：半透明 + 文字删除线 + 复选框填充渐变色

#### 任务操作
- 点击复选框：切换完成/未完成状态（乐观更新）
- 双击任务文本 / 点击编辑按钮：弹出编辑弹窗，可修改内容、优先级、分类、截止日期
- 点击删除按钮：确认弹窗 → 删除（乐观更新）→ Snackbar 显示撤销按钮
- 撤销删除：点击 Snackbar 撤销按钮（乐观更新）
- 清除已完成：底部按钮，确认后批量删除（乐观更新）

#### 底部统计
- 显示"共 X 项，已完成 Y 项"

### 2.4 统计 Tab

- 总体完成率卡片：白色背景 + SVG 圆形进度环 + 百分比 + "X / Y 完成"
- 双格卡片：连续完成天数（🔥）+ 待完成数（⏳）
- 近 7 天完成趋势：柱状图（暖色渐变），按创建日期统计已完成任务数，显示星期标签
- 本月统计三格卡片：本月总计任务数 + 本月完成数 + 月均完成率

### 2.5 个人中心 Tab

- 渐变背景用户信息卡片（GitHub 头像 + 昵称 + 邮箱）
- 无头像时隐藏头像元素
- 设置区：深色/浅色模式切换
- 数据管理区：导出数据（JSON 文件下载）/ 导入数据（JSON 文件上传）
- 退出登录按钮（红色标识）

## 3. 导航结构

| 位置 | 方式 | 说明 |
|------|------|------|
| 顶部 | 粘性 Tab 导航栏 | 4 个 Tab：首页 / 任务 / 统计 / 我的 |
| 激活态 | 渐变下划线 + 主色文字 | 当前 Tab 高亮 |
| 切换动画 | 淡入 + 上移 | 页面内容 fadeIn 动画 |

## 4. 设计规范

### 4.1 色彩体系

| 用途 | 浅色模式 | 深色模式 |
|------|----------|----------|
| 主色 | #FF6B6B | #FF6B6B（不变） |
| 渐变 | #FF6B6B → #FF8E53 | #FF6B6B → #FF8E53 |
| 背景 | #F8F5F2 | #1A1A2E |
| 卡片 | #FFFFFF | #16213E |
| 正文 | #2D3436 | #EAEAEA |
| 次要文字 | #636E72 | #A0A0B0 |
| 边框 | #F0EDED | #2A2A40 |

### 4.2 圆角与阴影

- 大卡片：20px 圆角
- 普通卡片：16px 圆角
- 输入框/按钮：10px 圆角
- 标签/胶囊：10-20px 圆角
- 阴影：rgba(255, 107, 107, 0.08) 浅色 / rgba(0, 0, 0, 0.2) 深色

### 4.3 动效

- 页面切换：0.3s fadeIn + translateY
- SVG 进度环：0.8s ease 过渡
- 按钮 hover：scale(1.05)
- 卡片 hover：translateY(-2px)
- Snackbar：bottom 0.3s 滑入

## 5. 交互方式

| 操作 | 桌面端 | 移动端 |
|------|--------|--------|
| 编辑任务 | 双击文本 或 点击编辑图标 | 点击编辑图标 |
| 完成任务 | 点击复选框 | 点击复选框 |
| 删除任务 | 点击删除图标 | 点击删除图标 |
| 添加任务 | Enter 或 点击"+" | 点击"+" |

## 6. 非功能需求

| 类别 | 要求 |
|------|------|
| 响应速度 | 所有操作乐观更新，UI ≤100ms 响应；失败自动回滚并提示 |
| 适配性 | 移动端 ≤480px / 桌面端 ≥768px 自适应 |
| 深色模式 | CSS 变量覆盖，支持手动切换 + 自动检测系统偏好 |
| 数据安全 | Supabase RLS 行级安全策略，每用户数据隔离 |
| 错误处理 | 网络失败自动回滚 + Snackbar 错误提示 |
| 浏览器兼容 | Chrome / Edge / Safari / Firefox 最近 2 个主版本 |

## 7. API 接口

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /api/todos | 获取任务列表（支持 completed/category/search/due_today 筛选） |
| POST | /api/todos | 创建任务 |
| PUT | /api/todos/:id | 更新任务 |
| DELETE | /api/todos/:id | 删除任务 |
| PUT | /api/todos/reorder | 批量更新排序 |
| DELETE | /api/todos/completed | 清除所有已完成 |
| GET | /api/todos/export | 导出全部数据 |
| POST | /api/todos/import | 批量导入任务 |

所有接口需要 Authorization: Bearer token 认证。

## 8. 数据模型

### todos 表

| 字段 | 类型 | 说明 |
|------|------|------|
| id | uuid | 主键 |
| content | text | 任务内容 |
| completed | boolean | 是否完成 |
| priority | text | 优先级（high/medium/low） |
| category | text | 分类 |
| due_date | timestamp | 截止日期 |
| order | integer | 排序序号 |
| user_id | uuid | 所属用户 |
| created_at | timestamp | 创建时间 |
